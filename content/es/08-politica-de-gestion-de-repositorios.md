# 📜 Política de Gestión de Repositorios | AI4Devs 2026/06 Seniors

**1\. Objetivo**  
Establecer lineamientos claros para la gestión de repositorios de proyectos dentro del programa IA4Devs, equilibrando la visibilidad profesional de los alumnos con la protección de información sensible.

**2\. Clasificación de repositorios**

**2.1 Repositorios públicos**  
Los repositorios pueden ser públicos cuando cumplan con las siguientes condiciones:

-   El proyecto es presentable como parte del portfolio profesional del alumno
    
-   No contiene información sensible o confidencial
    
-   Incluye documentación adecuada (README) que explique el propósito y funcionamiento del proyecto
    

**2.2 Repositorios privados**  
Se recomienda mantener repositorios privados en los siguientes casos:

-   El proyecto contiene credenciales, tokens o claves de acceso
    
-   Se utilizan APIs reales o servicios con datos sensibles
    
-   Existe información interna o no destinada a difusión pública
    
-   El proyecto no está preparado para ser compartido externamente
    

**3\. Buenas prácticas obligatorias**  
Para todos los repositorios, especialmente los públicos, se deben cumplir las siguientes prácticas:

-   No incluir archivos .env en el repositorio
    
-   No exponer claves API, tokens, contraseñas o credenciales
    
-   No incluir datos reales de usuarios o información personal
    
-   Utilizar archivos .env.example para documentar variables necesarias
    
-   Emplear valores ficticios o de prueba en configuraciones compartidas
    
-   Incluir un README con instrucciones claras de instalación y uso
    

**4\. Responsabilidad**  
Cada alumno es responsable del contenido que publica en sus repositorios.  
IA4Devs no se hace responsable por la exposición de información sensible derivada de un uso inadecuado de estas prácticas.

**5\. Enfoque de visibilidad**  
La publicación de repositorios como proyectos públicos forma parte del desarrollo profesional de los alumnos, facilitando la creación de portfolio y la exposición en el ecosistema tecnológico.

Se recomienda evaluar de forma consciente qué proyectos compartir públicamente y bajo qué condiciones.

  
**6\. Recomendaciones adicionales**

-   Revisar el repositorio antes de hacerlo público
    
-   Utilizar herramientas de escaneo de secretos cuando sea posible
    
-   Mantener actualizado el repositorio con buenas prácticas de documentación
    
-   En caso de duda, optar por mantener el repositorio privado
